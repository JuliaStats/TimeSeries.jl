Modify existing TimeArrays

Modify existing TimeArrays

Since TimeArrays are immutable, they cannot be altered or changed in-place. In practical application, an existing TimeArray might need to be used to create a new one with many of the same values. This might be thought of as changing the fields of an existing TimeArray, but what actually happens is a new TimeArray is created. To allow the use of an existing TimeArray to create a new one, the update and rename methods are provided.

update

The update method supports adding new observations only. Older and in-between dates are not supported:

julia> using TimeSeries

julia> using MarketData
ERROR: ArgumentError: Package MarketData not found in current path:
- Run `import Pkg; Pkg.add("MarketData")` to install the MarketData package.

julia> update(cl, Date(2002,1,1), 111.11)
ERROR: UndefVarError: cl not defined

julia> update(cl, Date(2002,1,1), [111.11])
ERROR: UndefVarError: cl not defined

julia> update(ohlc, Date(2002,1,1), [111.11 222.22 333.33 444.44])
ERROR: UndefVarError: ohlc not defined

rename

The rename method allows the column name(s) to be changed:

julia> using TimeSeries

julia> using MarketData
ERROR: ArgumentError: Package MarketData not found in current path:
- Run `import Pkg; Pkg.add("MarketData")` to install the MarketData package.

julia> rename(cl, :Close′)
ERROR: UndefVarError: cl not defined

julia> rename(cl, [:Close′])
ERROR: UndefVarError: cl not defined

julia> rename(ohlc, [:Open′, :High′, :Low′, :Close′])
ERROR: UndefVarError: ohlc not defined
